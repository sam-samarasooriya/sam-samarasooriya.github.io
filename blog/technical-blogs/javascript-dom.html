JavaScript Fundamentals

  • JavaScript and its relationship to HTML and CSS

    JavaScript (aka JS) is one of the most famous programming languages in the world. JS was introduced in 1995 and had significant growth and is now commonly used in web pages and apps in the present world.

    JavaScript language is a high-level, versatile programming language mainly used for creating interactive and dynamic features on websites. It allows developers to add or modify the content and behaviour of web pages in real-time with enhancing the user experience.

    As discussed in previous blog posts, HTML (Hyper Text Markup Language) provides the structure and content of a web page, such as text, images, and links. CSS (Cascading Style Sheets) handles the presentation, including layout, colours, and fonts. JavaScript ties it all together by adding interactivity. For example, while HTML defines a button and CSS styles it, JavaScript can control what happens when the button is clicked, such as showing a popup or changing content on the page.

  • Description of Control flow and loops, arrays, and objects

    Control flow determines the order in which statements are executed in a program. It includes

    Control Flow

    Control flow determines the order in which statements are executed in a program. It includes:

    1. Conditional Statements: Direct execution based on conditions (e.g., if, else, switch).
    2. Branching Statements: Alter the flow of execution (e.g., return, break, continue).

    Loops

    Loops are used to repeat a block of code multiple times:

    1. for Loop: Executes code a specific number of times (e.g., iterating through a range).
    2. while Loop: Repeats code while a condition is true.
    3. do...while Loop: Similar to while, but guarantees at least one execution of the code.

    Arrays

    Arrays are ordered collections of elements, usually of the same type, indexed by numbers:

    1. Creation: Defined with square brackets (e.g., let fruits = ['apple', 'banana']).
    2. Access: Elements are accessed by their index (e.g., fruits[0]).

    Objects

    Objects are collections of key-value pairs used to represent more complex data structures:

    1. Creation: Defined with curly braces (e.g., let person = { name: 'John', age: 30 }).
    2. Access: Properties are accessed using dot or bracket notation (e.g., person.name or person['age']).

    These concepts are fundamental to controlling the flow of execution and organizing data in programming.

  • Functions and why they are useful

    Functions are blocks of reusable code designed to perform a specific task. They help modularize and organize code by allowing you to encapsulate logic into discrete units

      Why Functions are Useful

    • Reusability: Once defined, functions can be called multiple times, reducing code duplication.
    • Modularity: Functions help break down complex problems into smaller, manageable pieces.
    • Maintainability: Encapsulating functionality in functions makes code easier to update and debug.
    • Abstraction: Functions allow you to hide implementation details and expose only necessary interfaces.
    • Organization: They help in structuring code logically, improving readability and collaboration.

  • The DOM and DevTools

    The DOM (Document Object Model)

    The DOM is a programming interface for web documents. It represents the structure of a webpage as a tree of objects, where each node corresponds to an element or attribute in the document.

    • Structure: The DOM maps HTML elements to objects that can be manipulated using JavaScript.
    • Manipulation: JavaScript can interact with the DOM to modify the content, structure, or style of a webpage dynamically
    • Events: The DOM allows you to handle user interactions (e.g., clicks, input) by attaching event listeners to elements.

    DevTools (Developer Tools)

    DevTools are a set of tools built into web browsers that help developers inspect, debug, and optimize web pages. Key features include:

    • Elements Panel: View and modify the HTML and CSS of a webpage in real-time.
    • Console: Log messages, run JavaScript code, and view errors and warnings.
    • Network Panel: Monitor network requests, view loading times, and debug API calls.
    • Sources Panel: Debug JavaScript code, set breakpoints, and step through code execution.
    • Performance Panel: Analyze and optimize page performance by profiling runtime behavior.
    • Application Panel: Inspect and manage web storage, cookies, and service workers.
